   draggr is a drag-and-drop personal scheduling tool app built in Angular. Unlike other scheduling tools, the tasks on your calendar are completely malleable: resizable, moveable, with tasks moving themselves automagically when needed. Rescheduling one task doesn’t mean rescheduling the whole day!

   I focused on creating a smooth UI that, with further development, will lend itself to gamification of users productivity.

   Currently, I consider it in an early-alpha stage. I’ve made it open-source, and plan on developing a whole array of features in the future. Very much aiming towards something I would find useful for daily use myself.

What I learned:

   I got a grasp on using my first MV* framework, learning a little about the front-end ecosystem along the way. From a workflow and logic standpoint, I found working in Angular to have more in common with making games using C# in Unity than any previous web-related work.

   It seems modern web development is abstracting a lot of logic from the back-end to the front-end – coincidentally, draggr is practically a fully-functioning app without ANY back-end besides a simple Firebase to store user data (for now). This is what got me hooked on front-end, and why I’ve been focusing my learning in this area.
